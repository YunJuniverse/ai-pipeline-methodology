# 프로젝트 킥오프 프롬프트 템플릿

> 새 프로젝트 시작 시 Claude Code에 붙여넣기.
> 모든 항목을 채울 필요 없음 — 아는 것만 채우고 나머지는 비워두면 AI가 질문함.

---

## 사용법

1. `./init-project.sh <이름> --type <유형>` 으로 프로젝트 폴더 생성
2. `cd <프로젝트폴더>` → `claude` 실행
3. 아래 템플릿을 채워서 첫 메시지로 전달

---

## 템플릿

```
CLAUDE.md를 읽고 프로젝트를 세팅해줘.

## 아이디어
[한 줄 설명: 무엇을 만들려고 하는가]

## 배경
- 왜 이걸 하려고 하는가: [동기/기회]
- 현재 상태: [아이디어만 / 리서치 완료 / 기획 진행 중 / 코딩 중]

## 타겟
- 핵심 사용자: [누구를 위한 서비스인가]
- 사용자의 문제: [이 사용자가 겪는 구체적 문제]

## 레퍼런스
- [서비스명 1] — [참고할 점]
- [서비스명 2] — [참고할 점]
- [서비스명 3] — [참고할 점]

## 차별점
[레퍼런스 대비 우리가 다르게 할 점 — 없으면 비워두고 같이 찾자]

## 제약 조건
- 예산: [없음 / 소규모 / 있음]
- 기간: [급함 / 여유 / 마감일 있음 → YYYY-MM-DD]
- 기술: [선호 스택 / 제약 / 모르겠음]
- 기타: [1인 개발 / 외주 예정 / 팀 있음]

## 오늘 할 일
[구체적 요청: 예) "Foundation 개요부터 잡아줘" / "사업기획서 초안 써줘"]
```

## 프로젝트 유형별 예시

### 예시 1: fullstack (기획 → 코딩)

```
CLAUDE.md를 읽고 프로젝트를 세팅해줘.

## 아이디어
남성 그루밍 정보를 공유하는 커뮤니티 플랫폼

## 배경
- 왜: 남성 그루밍 시장이 커지는데 정보가 파편화되어 있음
- 현재 상태: 아이디어 단계

## 타겟
- 핵심 사용자: 20~30대 남성, 피부/헤어 관리에 관심 있지만 어디서 정보를 얻을지 모르는 사람
- 사용자의 문제: 커뮤니티는 여성 중심, 남성 전용 정보 허브가 없음

## 레퍼런스
- 화해 — 리뷰 기반 제품 추천 구조
- Reddit r/malegrooming — 커뮤니티 운영 방식
- 오늘의집 — 콘텐츠+커머스 연결 모델

## 차별점
남성 전용 + 루틴 기반 추천 (단일 제품이 아닌 루틴 단위)

## 제약 조건
- 예산: 소규모 (1인 개발)
- 기간: 여유 (MVP 2개월 목표)
- 기술: Next.js + Supabase 선호
- 기타: 1인 개발

## 오늘 할 일
Foundation 초안부터 잡아줘. 시장성과 BM 방향을 먼저 정리하고 싶어.
```

### 예시 2: planning-only (기획서만)

```
CLAUDE.md를 읽고 프로젝트를 세팅해줘.

## 아이디어
영어 쉐도잉 학습 앱 — 원어민 음성을 따라하며 발음 교정

## 배경
- 왜: 영어 회화 앱은 많지만 쉐도잉 특화 앱은 부족
- 현재 상태: 리서치 완료, 기획 필요

## 타겟
- 핵심 사용자: 영어 중급자, 회화는 되지만 발음/억양에 자신 없는 직장인
- 사용자의 문제: 혼자 연습할 때 피드백이 없음

## 레퍼런스
- Elsa Speak — AI 발음 평가
- Speechling — 원어민 피드백 모델
- 리얼클래스 — 콘텐츠 기반 학습 구조

## 차별점
쉐도잉에 집중 + AI 실시간 발음 비교 피드백

## 제약 조건
- 예산: 투자 유치 검토 중
- 기간: 기획서 2주 내 완성
- 기술: 미정 (기획 먼저)
- 기타: 대표 보고용 사업기획서 필요

## 오늘 할 일
Foundation 완성본 써줘. 투자자/대표 보고용이라 상세하게.
```

### 예시 3: research (리서치만)

```
## 오늘 할 일
시장 규모와 규제 환경부터 정리해줘. Foundation 개요 수준으로.
```

### 예시 4: coding-only (기존 기획 기반 바로 코딩)

```
CLAUDE.md를 읽고 프로젝트를 세팅해줘.

## 아이디어
헤어제품 쇼핑몰 MVP 구현

## 배경
- 왜: 기획서는 완성됨, 이제 구현 단계
- 현재 상태: docs/foundation.md, docs/spec.md 작성 완료

## 제약 조건
- 기술: Next.js + Supabase + Stripe
- 기간: MVP 3주
- 기타: 1인 개발

## 오늘 할 일
spec.md 읽고 도메인 모델부터 설계해줘. Stage 4 시작.
```

## 항목별 가이드

| 항목 | 왜 필요한가 | 안 채우면? |
|------|------------|-----------|
| 아이디어 | AI가 프로젝트 방향을 잡는 출발점 | **필수** — 이것 없으면 시작 불가 |
| 배경 | 동기와 현재 상태로 시작 Stage 판단 | AI가 질문함 |
| 타겟 | Foundation의 고객 정의에 직접 반영 | AI가 일반적인 타겟으로 시작 |
| 레퍼런스 | 포지셔닝과 차별점 설계의 기준선 | AI가 자체 리서치 (품질 하락) |
| 차별점 | Foundation의 핵심 — 없으면 문서가 일반적 | AI와 같이 찾는 것도 OK |
| 제약 조건 | 스택, 일정, 예산이 모든 결정에 영향 | AI가 과도한 스펙 제안 가능 |
| 오늘 할 일 | Stage와 깊이 결정 | AI가 Stage 0부터 시작 |

> **핵심: 아이디어 + 레퍼런스 + 오늘 할 일, 이 3개만 있어도 시작 가능.**
> 나머지는 AI가 물어보거나, 작업하면서 채워진다.
