# Service Spec Snapshot Prompt

Use this when a dated service overview or scoped feature spec is needed.

## Instructions

- Read `HANDOFF.md`, relevant TODO items, relevant PRs, relevant ADRs, and code only for the scope being described.
- Keep the scope explicit. State what is out of scope.
- Do not treat this document as the live source of truth after creation.
- Write the result to `docs/snapshots/service-spec-YYYY-MM-DD.md`.
- Add a header that says the document is a snapshot and not a live source.

## Include

- scope
- user outcomes
- acceptance criteria
- major flows
- dependencies
- risks
- evidence links
