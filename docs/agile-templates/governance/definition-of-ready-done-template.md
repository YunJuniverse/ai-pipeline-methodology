# Definition Of Ready / Done Template

> Version: v0.1 | Date: YYYY-MM-DD

## 1. Purpose
- This document defines when work is ready to start and when it is done enough to review or release.

## 2. Definition Of Ready

### Story / Feature Ready Checklist
- [ ] Related planning docs exist
- [ ] Related development spec exists
- [ ] Acceptance criteria are explicit
- [ ] Out-of-scope is explicit
- [ ] Dependencies are identified
- [ ] Design or wireframe references are attached if needed
- [ ] Test approach is known
- [ ] Human owner is assigned
- [ ] Approval to start build is given

## 3. Definition Of Done

### Build Done Checklist
- [ ] Scope implemented
- [ ] Tests added and passing
- [ ] Lint and type checks pass
- [ ] Error handling included
- [ ] Human-readable code review completed
- [ ] Relevant ADR updated
- [ ] Known issues documented
- [ ] Sprint report updated

### Release Done Checklist
- [ ] QA complete for target scope
- [ ] UI review complete
- [ ] Critical defects resolved or explicitly accepted
- [ ] Rollback or mitigation plan exists
- [ ] Stakeholder approval complete
