# Standard Prompts for AI Relay

> 텍스트 익스팬더 또는 키보드 단축키로 등록해서 사용.

---

## 1. 프로젝트 시작 프롬프트 (Claude 전용)

```
새 프로젝트를 시작합니다.

[프로젝트 목표와 요구사항을 여기에 작성]

작업 완료 후, 아래 핸드오프 템플릿에 따라 핸드오프 파일도 함께 생성해주세요.

### 핸드오프 템플릿 구조:
- OBJECTIVE (불변): 프로젝트 최종 목표
- KEY DECISIONS SO FAR: 이번 작업에서 확정한 사항
- WHAT CHANGED THIS ROUND: 이번에 만든 내용 요약
- OPEN ISSUES: 미해결 이슈 (테이블 형태, AI 추천 포함)
- NEXT AGENT ROLE: 다음 AI의 역할, 작업, 집중 포인트
```

---

## 2. 릴레이 표준 프롬프트 (모든 AI 공통)

```
첨부된 핸드오프 파일과 산출물을 읽고, NEXT AGENT ROLE에 명시된 작업을 수행해주세요.

작업 완료 후, 동일한 핸드오프 포맷으로 새 핸드오프 파일을 생성해주세요.
핸드오프에는 반드시 다음을 포함해주세요:
- OBJECTIVE는 그대로 유지
- KEY DECISIONS에 새 결정사항 추가
- WHAT CHANGED에 이번 변경분 기록
- OPEN ISSUES 테이블 업데이트 (해결/이월/신규 추천)
- NEXT AGENT ROLE에 다음 단계 지시
```

---

## 3. 리뷰어 특화 프롬프트 (피어리뷰 단계)

```
첨부된 핸드오프 파일과 산출물을 읽고, NEXT AGENT ROLE에 명시된 리뷰 작업을 수행해주세요.

리뷰는 아래 구조로 작성해주세요:
1. 강점 (잘 된 부분)
2. 약점 (개선이 필요한 부분)
3. 구체적 제안 (어떻게 고치면 좋을지)
4. 치명적 빠짐 (놓친 관점이나 누락된 내용)

작업 완료 후, 동일한 핸드오프 포맷으로 새 핸드오프 파일을 생성해주세요.
```

---

## 4. 최종 산출물 프롬프트 (마지막 단계)

```
첨부된 핸드오프 파일과 산출물을 읽고, NEXT AGENT ROLE에 명시된 최종 작업을 수행해주세요.

이번이 릴레이의 마지막 단계입니다.
OPEN ISSUES가 모두 해결되었는지 확인하고, 최종 산출물을 완성해주세요.

핸드오프 파일 대신, 프로젝트 완료 요약을 작성해주세요:
- 최종 결과물 설명
- 전체 릴레이에서의 주요 변경 히스토리
- 남은 후속 작업 (있다면)
```

---

## 사용 팁

- 대부분의 경우 **#2 릴레이 표준 프롬프트**만으로 충분합니다
- 파일 첨부 + 표준 프롬프트 복붙 = 10초 이내 릴레이 전달
- 핸드오프가 짧으면 파일 첨부 대신 본문에 복붙해도 OK
