# Agile Deliverables Map

This methodology uses a repeating sprint loop instead of a single linear pass.

## Required Output Chain

1. Research output
2. Planning docs set
3. Development spec
4. MVP build
5. Sprint completion report
6. Sprint feedback report
7. Updated planning docs
8. Updated development spec
9. Next MVP build

## Planning Documents Set

- `service-overview`
- `requirements`
- `user-stories`
- `information-architecture`
- `user-flow`
- `wireframes`
- `functional-spec`
- `data-model`
- `operations-policy` (recommended)

## Versioning Example

| Sprint | Planning Docs | Development Spec | Build | Reports |
|--------|---------------|------------------|-------|---------|
| 01 | v0.1 | v0.1 | v0.1 | sprint-01 |
| 02 | v0.2 | v0.2 | v0.2 | sprint-02 |
| 03 | v0.3 | v0.3 | v0.3 | sprint-03 |

## Human Approval Gates

- Approve initial planning set before first build
- Approve sprint completion before closing the sprint
- Approve feedback-driven revisions before next sprint build
